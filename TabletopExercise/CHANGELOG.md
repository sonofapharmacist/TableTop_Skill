## 2026-04-10

### Bug Fixes

#### `README.md` — Added Client Environment Scoping section
- **Added:** "Before You Start — Client Scoping" section at the top of Quick Start, covering platforms, crown jewels, regulatory context, participants, and constraints.
- **Purpose:** Reference for exercise designers before a client engagement — no need to invoke Claude just to recall what to ask.
- **Note:** Section includes the same guidance as SKILL.md: these are internal research questions, not a client-facing questionnaire.

#### `SKILL.md` — Added Client Environment Scoping section
- **Added:** New "Client Environment Scoping" section between Initial Setup Questions and Output Formats.
- **Purpose:** Provides Claude with structured background research questions to collect before designing a tailored exercise — covering platforms, crown jewels, regulatory context, participants, and constraints.
- **Design intent:** Framed as internal research for Claude, not a client-facing questionnaire. Detection, backup, and identity gaps are explicitly noted as content that should surface organically during the exercise rather than be pre-disclosed to participants.
- **Impact on generation:** Answers feed directly into `scenarioOverview`, `attackVector`, `potentialImpact`, and `gapAnalysis` JSON fields.

#### `SKILL.md` — Added `includeEmoji` to Initial Setup Questions
- **Added:** Second setup question: *"Include emoji icons in headings and labels? (default: No)"* asked once per session alongside `preparedBy`.
- **Behaviour:** Yes → pass `--emoji` to the bun runner (or set `includeEmoji: true`). No/skipped → omit flag (default, no emoji).
- Both setup questions are now asked together before any exercise is generated, and carried through the session.

#### `generate-html.ts` — Options object refactor + `includeEmoji` flag
- **Problem:** `generateStandaloneHTML` had grown to multiple positional boolean parameters (`isFacilitator`, and now `includeEmoji`), which is fragile and hard to read at call sites.
- **Fix:** Replaced positional booleans with a `GenerateOptions` interface `{ isFacilitator?: boolean; includeEmoji?: boolean }`. Both default to `true` for backwards compatibility. Added `const e = (emoji: string) => includeEmoji ? emoji + ' ' : ''` helper inside the function; all 28 emoji occurrences in the template now use `${e('...')}` instead of literal emoji characters.
- **Both runners** updated to `{ isFacilitator: true/false, includeEmoji: false }` — emoji off by default for client-facing output.
- **Follow-up:** Initial grep pattern missed 9 emoji (`🎤 ⚖️ 📊 🔧 🛠️ ❓ 📈`) in the Facilitator Guide and gap analysis sections. A unicode-range Python scan confirmed all occurrences are now wrapped. Total: 37 emoji sites gated.
- **Files changed:** `generators/generate-html.ts`, `generators/run-heartland-foods.ts`, `generators/run-ransomware-hybrid.ts`.

#### `generate-html.ts` / `run-heartland-foods.ts` / `run-ransomware-hybrid.ts` — Timeline indentation in participant HTML
- **Problem:** `stripFacilitatorNotes` post-processing regex was consuming the closing `</div>` of each `timeline-event` along with the facilitator notes block, leaving every timeline event unclosed. The browser interpreted the unclosed elements as nested, cascading further right with each step.
- **Fix:** Added `isFacilitator: boolean = true` parameter to `generateStandaloneHTML`. All inline facilitator content (timeline notes, objective notes, inject notes, atomic notes, gap notes, expected responses) is now gated at render time. The participant version is generated by calling `generateStandaloneHTML(data, false)` directly, eliminating the need for `facilitator-notes` div regex entirely. Section-level regex (facilitator-guide section, atomics section, nav links, banner) is retained in both runners as those target whole `<section>` elements with no nesting ambiguity.
- **Files changed:** `generators/generate-html.ts` (signature + 6 render sites), `generators/run-heartland-foods.ts` (replaced `stripFacilitatorNotes`), `generators/run-ransomware-hybrid.ts` (replaced stripping logic).

#### `generate-html.ts` / `generate-pdf.ts` — Apostrophes rendering as `&#039;`
- **Problem:** The `escapeHtml` function in both generators included `.replace(/'/g, '&#039;')`, causing all apostrophes in exercise content to render as the literal string `&#039;` in the browser.
- **Fix:** Removed the apostrophe replacement line from `escapeHtml` in both files. Encoding `'` as `&#039;` is only necessary inside unquoted HTML attributes; all content in these generators is placed in element body text where a literal apostrophe is valid.
- **Files changed:** `generators/generate-html.ts` (line 12), `generators/generate-pdf.ts` (line 124).

#### `run-heartland-foods.ts` / `run-ransomware-hybrid.ts` — Atomics stripped from participant output
- **Problem:** Both exercise runners were generating participant HTML files that still contained the full Technical Atomics Runbook section and its nav link, exposing runner instructions to participants.
- **Fix:** Added regex stripping of `<section id="atomics">` and `<a href="#atomics">` nav link to `stripFacilitatorNotes` in both runners.
- **Verified:** Participant files now contain 0 atomic references; facilitator files unchanged.

#### `generate-html.ts` — Collapsible card click behavior
- **Problem:** `onclick="this.classList.toggle('open')"` was attached to the entire `.collapsible-card` div, causing any click inside the expanded body (text selection, form inputs, code blocks) to collapse the card.
- **Fix:** Moved the `onclick` handler to `.collapsible-header` only, using `this.parentElement.classList.toggle('open')`.
- **Affected sections:** Exercise Injects (line 1314) and Technical Atomics Runbook (line 1420).

- **Problem:** Three generator path references used `~/.openclaw/workspace/skills/TabletopExercise/generators/` (old install location). One additional reference used a hardcoded `/root/.claude/...` path in the PDF regeneration section.

  This derives the install path at runtime, so future path changes require updating only one location.
