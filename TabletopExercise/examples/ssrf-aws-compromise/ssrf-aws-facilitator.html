01

Executive Summary

This tabletop exercise simulates a critical security incident where a threat actor exploits an SSRF vulnerability in the organization's e-commerce web application to access the AWS EC2 instance metadata service. The attacker successfully steals temporary IAM role credentials and uses them to access sensitive S3 buckets containing customer PII.

The scenario tests the organization's ability to detect SSRF exploitation, respond to AWS credential compromise, contain the breach, and implement cloud security controls.

02

Scenario Overview

Your organization runs a Flask-based e-commerce platform on AWS EC2. The application includes a product price lookup feature that fetches pricing from external APIs. A developer introduced an SSRF vulnerability by using user-supplied input in requests.get() without validation.

# Vulnerable Code
@app.route('/api/price')
def get_price():
    url = request.args.get('url')
    response = requests.get(url)  # No validation!
    return response.text

SOP/Playbook Gap Analysis

SSRF Incident Response PlaybookMissing

No documented procedure for SSRF-specific response. Need to create playbook including metadata service blocking procedures.

AWS Credential Rotation ProcedurePartial

No documented procedure for emergency credential rotation. Need automation for credential rotation.

Data Breach Notification ProcedurePartial

Unclear who makes notification decision. Need templates and clear regulatory timeline.

EC2 IMDS ConfigurationMissing

IMDSv2 not enforced. Need to enable on all instances and implement security group rules blocking metadata IP.
